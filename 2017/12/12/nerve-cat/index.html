CreateJS实战 "围住神经猫"

发表于 2017-12-12

一、概述

CreateJS是一套便于HTML5开发游戏的库，当前这个实战例子主要用到其中之一的EaselJS(主要处理canvas)

二、场景搭建

①，确定游戏场景尺寸

游戏场景尺寸 → canvas画布的大小
通过素材的大小大概拟定一下就好了，宽高 => (480 * 800)

②，背景图片处理

  • 第一种方式：背景图处理，直接img标签 | background-image属性就可以了
  • 第二种方式：js new Image 生成图片 → 丢进canvas绘制

四、创建动画

动画原理：每秒播放不同的图片，形成视觉残留，错觉上感觉是连续的
动画中的fps是每秒播放多少帧图片，和游戏上有些差异，游戏中的fps指刷新频率，每秒刷新多少次

纹理集动画(SpriteSheet)

实际上就是常说的精灵图，将多张图片拼接到一张大图上，一次性导入和管理。
不同的是，通常会附带一个描述文件，说明是如何播放动画的(xml、json)

用EaselJS实现纹理集动画

  1. 改写纹理集动画的描述，以EaselJS的格式。

  2. 实现动画

五、创建地图

①，分析地图

神经猫的地图很明显是一个二维数组，只不过稍微有些变形、交错。

②，对地图每一个坐标的圆进行抽象 → 类

原因是：神经猫的逃跑要根据每一个坐标上圆是否可以被选中来决定，也就是说，每一个圆至少要有坐标值信息以及当前选中状态信息。显然，面向对象的方式会简便很多。

③，使用自定义类进行绘制地图

④，简单随机算法 → 初始地图

六、监听点击、判断点击圆

①. 监听点击事件

给每一个圆都绑定监听事件，这样就很容易判断点击圆，如果用委托代理做绑定的话，还要循环判断e.target是否在数组内，比较麻烦。

需要注意：

  • 报错信息：画布像素跨域访问出错
  • 解决：放到服务器上访问就可以了，不要本地打开index.html
  • 服务器搭建：最简单的就是nodejs, 安装命令：npm install http-server -g, 目标目录下启动：http-server, local:8080就可以访问了

②，点击切换圆的状态

Circle.TYPE_SELECTED → Circle.TYPE_UNSELECTED

模拟切换：

  • a.移除当前位置的圆
  • b.新增一个Circle.TYPE_UNSELECTED的圆
  • c.新增后一定不要忘记添加到stage中，否则无法显示
  • d.显示需要注意，在原位置显示，所以移除之前需要保留x,y坐标值
